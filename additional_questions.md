# תשובות לשאלות להעמקה

## 1. שיקולי זיכרון לרכיב Embedded (C++/Assembly)
במעבר לרכיב Embedded מוגבל במשאבים, הייתי לוקח בחשבון:
- **זיכרון דינמי:** הימנע משימוש ב-new/delete או malloc/free כדי למנוע דליפות זיכרון. השתמש במערכים סטטיים או pools מוגדרים מראש.
- **גודל מבני נתונים:** השתמש בטיפוסים קטנים יותר (uint8_t במקום int) ודחוס נתונים ככל האפשר.
- **חישובים מצטברים:** במקום לאחסן רשימה של כל הערכים, חשב סכומים וממוצעים באופן מצטבר כדי לחסוך זיכרון.
- **זיכרון קבוע:** שמור קבועים ב-ROM במקום RAM.
- **אופטימיזציה של לולאות:** וודא שלולאות לא יוצרות overhead מיותר.

## 2. אבטחת API Endpoint
לאבטחת ה-API שחושף נתוני מכשירים:
- **אימות:** השתמש ב-JWT או OAuth2 עם refresh tokens.
- **הרשאות:** יישם Role-Based Access Control (RBAC) כדי לוודא שמשתמשים יכולים לגשת רק לנתונים שלהם.
- **הצפנה:** השתמש ב-HTTPS עם TLS 1.3, והצפן נתונים רגישים ב-DB.
- **Rate Limiting:** מנע DDoS עם מגבלות על מספר בקשות.
- **Validation:** וודא input validation כדי למנוע SQL injection או XSS.
- **Logging:** תעד גישה לנתונים רגישים לצורך audit.

## 3. שאלות למנהל המוצר לפני פיתוח AI Insights

- איזה סוג של AI נדרש (ML, NLP, או משהו אחר)?
- מהו מקור הנתונים לניתוח AI?
- איך נמדוד את הצלחת הרכיב (metrics כמו דיוק, זמן תגובה)?
- האם יש מגבלות על שימוש בנתונים (privacy, GDPR)?
- מה קורה אם ה-AI לא יכול לייצר תובנה (fallback mechanism)?
- איך הרכיב ישתלב בתהליך העבודה הקיים של המשתמשים?
